Minhas Habilidades

Tenho uma vasta experiência e um conjunto diversificado de habilidades na área de Testes de Software, abrangendo desde testes manuais até testes automatizados e de performance. Aqui estão algumas das minhas principais competências:

Testes Manuais

  • Planejamento e execução de testes manuais detalhados.
  • Identificação e documentação de defeitos.
  • Testes de regressão para garantir a estabilidade do software após mudanças.
  • Habilidade para definir e documentar as melhores práticas de teste para cenários específicos.

Testes Automatizados

Automação de Testes de API

  • Utilizando Mocha, Chai, Node.js, JavaScript e POSTMAN para criar e manter testes automatizados de APIs REST.

Automação de Testes de Frontend

  • Experiência em criar testes automatizados de interfaces de usuário utilizando Cypress, garantindo a funcionalidade e a usabilidade das aplicações web.

Testes de Performance

  • K6: Execução de testes de performance para avaliar a capacidade de resposta e a estabilidade do sistema sob carga. Criação de scripts de teste de carga e análise dos resultados para identificar gargalos e otimizar o desempenho.

Outras Competências

  • Selenium Conhecimento básico na automação de testes de interface com Selenium WebDriver.
  • JUnit e Cucumber: Experiência em testes baseados em comportamento (BDD) com JUnit e Cucumber.
  • Docker:Utilização de Docker para criar ambientes de teste isolados e consistentes.

Ferramentas e Tecnologias

  • Git: Controle de versão e colaboração em projetos usando Git.
  • Mocha e Chai: Ferramentas preferidas para testes de unidade e de integração em projetos Node.js.
  • JavaScript: Linguagem principal para a automação de testes e desenvolvimento de scripts de teste.
  • Postman: Ferramenta usada para testes de API manuais e automação de testes de API.

Competências Adicionais

  • Metodologias Ágeis: Experiência com Scrum e entrega incremental de funcionalidades através de Sprints.
  • Definição de Estratégias de Teste: Desenvolvimento de estratégias abrangentes de teste, tanto manuais quanto automatizados, para assegurar a qualidade do software.
  • Documentação: Habilidade para definir e documentar as melhores práticas de teste para diversos cenários.
  • Análise de Riscos: Identificação e análise de riscos relacionados à qualidade do software, propondo medidas mitigadoras apropriadas
